- 정보관리에서 계열사 사용자의 권한에 낙찰권한을 선택할 수 있게 함.
  (사용자에 개찰자권한을 추가하는 것처럼 낙찰자권한을 추가할 수 있게끔 함)
- 입찰 생성 시 낙찰자를 선택할 때 낙찰자 권한을 가진 사용자만 조회되어 선택이 되게 함 

-----------입찰권한 정의-----------
1. 담당자 : 입찰을 생성하는 자
- 권한 : 입찰생성, 입찰공고, 유찰, 재입찰 => 개찰, 낙찰만 빼고 다 할 수 있음
2. 공고자
- 권한 : 입찰공고
3. 개찰자
- 권한 : 개찰, 낙찰,  유찰, 재입찰
4. 낙찰자
- 권한 : 낙찰, 유찰 => 낙찰자는 재입찰 권한이 없음
5. 입회자
- 권한 : 읽기만 가능
